Rede para aceder ao NeoVision

No mesmo PC onde corres o INICIAR-API.cmd ou o uvicorn: abre no browser http://127.0.0.1:PORTA/. A porta aparece na janela preta quando o servidor inicia (8080 habitual; 9080 se 8080 estiver ocupada).

Este separador já usa o anfitrião e porta corretos, desde que abras o link a partir dessa máquina. Para testar rápido: entra primeiro no painel e depois em Sistema.

Outro equipamento na mesma LAN (telefone, segundo PC): o arranque padrão da API só escuta em 127.0.0.1, pelo que esse outro não consegue ligar até passares o servidor para aceitar a rede interna:

  • Arranca o uvicorn com --host 0.0.0.0 (e a mesma --port). No outro equipamento usa http://IP-DESTE-PC:PORTA/painel — descobre o IP em Definições de rede do Windows ou ipconfig.
  • No Firewall do Windows, permite entrada TCP para essa porta só na rede privada (ou apenas para IPs de confiança). Não expor a Internet diretamente sem VPN ou HTTPS reverso.

Base de dados (MySQL) quando não está neste mesmo PC — variáveis de ambiente antes de iniciar a API (NEOVISION_MYSQL_HOST, porta, utilizador): vê os comentários no INICIAR-API.cmd e no NeoVision Desktop (appsettings.json).

IPs das câmeras — colocar no NeoVision

As câmeras ficam na tabela cameras (MySQL), partilhada entre a API e o NeoVision Desktop. Para o sistema “ver” cada câmera, precisas do endereço IP (ou hostname) que o PC do NeoVision consegue alcançar — em geral a mesma rede local (Wi-Fi ou cabo) que o servidor.

  • IP estável: no router, reserva DHCP para o MAC da câmera ou configura IP fixo no equipamento — senão o IP muda e o registo deixa de bater certo.
  • Formulário no painel: Adicionar câmera — mesmo POST /cameras com IP, portas, URL RTSP gerada a partir de utilizador/senha.
  • Ou pela API / Swagger: POST /cameras com JSON; ip_address obrigatório; name, http_port, rtsp_url, onvif_endpoint opcionais.
  • Registo pelo Desktop: «Procurar câmeras» (ONVIF) e «Guardar na base MySQL» — preenche a mesma tabela sem escritos manuais de JSON.
  • Depois: GET /cameras lista tudo com o id (UUID); GET /cameras/{id}/status testa RTSP/redes.

Firewall: o PC servidor tem de conseguir sair até à câmera nos portos que ela usa (HTTP, RTSP habitualmente).
