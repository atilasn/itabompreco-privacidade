NeoVision AI - API
==================

"Conexao recusada" (ERR_CONNECTION_REFUSED)
-----------------------------------------
O browser so funciona DEPOIS do servidor estar a correr.

1) Duplo clique nesta pasta:  INICIAR-API.cmd
   (ficheiro preparado para verificar Python, instalar pacotes e arrancar)

2) NAO FECHE a janela preta. Espere ver:
   "Uvicorn running on http://127.0.0.1:8080"  (ou :9080)

3) No browser, abra a PAGINA DE BOTOES (inicio com links):
     http://127.0.0.1:8080/painel
   (ajuste 8080 se a API usar outra porta, ex. 9080)
   Se quiser: http://127.0.0.1:8080/health

Se carregou "Continuar" num aviso do Windows (SmartScreen):
- e normal a primeira vez; use "Mais informacoes" e "Executar mesmo assim"
  se o ficheiro for o nosso INICIAR-API.cmd nesta pasta.

Outros atalhos: start-api.bat  |  start-api-9080.bat  |  start-api-auto.bat

Instalavel (executavel) — dist
-----------------------------
- A partir da raiz do repositorio:  powershell -ExecutionPolicy Bypass -File .\build\publish-api-windows.ps1
- Sera gerada  dist\NeoVision-API\NeoVision-API.exe  (pasta com ficheiros; nao e preciso Python no PC)
- Pode executar  INICIAR-API-DIST.cmd  nessa mesma pasta (definicoes MySQL em variaveis de ambiente)

Camaras (MySQL) e NEOVISION_MYSQL_*
------------------------------------
- O servico lê a tabela `cameras` (mesma base do app WPF). INICIAR-API.cmd define
  NEOVISION_MYSQL_HOST/PORT/USER/PASSWORD/DATABASE se ainda nao estiverem no Windows.
- Ajuste a palavra-passe se for diferente de "changeme" (ou defina a variavel
  NEOVISION_MYSQL_PASSWORD antes de arrancar).
- GET /cameras  |  GET /cameras/{uuid}/status  (veja o Swagger em /docs)
